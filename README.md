## 🧩 节点详解

### 1. 文本传递 (WuhuoTextGate)
**核心节点**。一个带有逻辑控制的文本处理节点，常用于 Prompt 的管理和工作流的调试。它有三种工作模式，通过 `enable_edit` 和 `free_pass` 开关组合控制。

#### 参数说明
*   `enable_edit` (Boolean): 开启编辑模式（红灯）。
*   `free_pass` (Boolean): 开启直通模式（绿灯）。
*   `in_text` (String): 上游输入的文本。
*   `edit_text` (String): 节点内手动输入的文本。
*   `key_word` (String): 固定前缀关键词。
*   `enhance_mode` (List): 预设的强化提示词（如“画质增强”），会自动添加到文本最前方。

#### 工作模式逻辑
1.  **🟢 直通模式 (Free Pass = True)**
    *   **场景**: 正常运行工作流，优先使用上游输入。
    *   **逻辑**:
        *   如果上游有输入 (`in_text`)，输出 = `前缀` + `上游文本`。
        *   如果上游无输入，输出 = `前缀` + `节点内编辑文本`。

2.  **🔴 编辑/截断模式 (Enable Edit = True)**
    *   **场景**: 调试时暂停工作流，或者修改上游传来的 Prompt。
    *   **逻辑**:
        *   工作流运行到此节点时会**自动暂停** (Interrupt)。
        *   节点会**捕获**上游传来的文本，并显示在节点上（需刷新或查看状态）。
        *   此时你可以修改 `edit_text`，准备进行下一次手动发送。

3.  **🟡 手动/混合模式 (Free Pass = False & Enable Edit = False)**
    *   **场景**: 忽略上游输入，强制使用节点内的文本；或者使用之前捕获的文本。
    *   **逻辑**:
        *   **最高优先级**: 如果 `edit_text` 有内容，直接输出 `前缀` + `edit_text` (忽略上游)。
        *   **次级**: 如果 `edit_text` 为空，但上游有新输入，输出 `前缀` + `上游文本`。
        *   **再次级**: 如果都为空，但之前在“红灯模式”下捕获过文本，输出 `前缀` + `捕获文本`。

### 2. 文本传递++ (WuhuoTextGatePro)
**文本传递的终极威力加强版**。保留了 `WuhuoTextGate` 所有的核心逻辑流控，但针对深度交互进行了革命性的前端 UI 重构。极力推荐在中大型复杂工作流中使用。

#### 🆕 核心进化亮点
*   **物理极客 HUD 控制台**：节点内置了一个完全响应式的交互式工具栏，不需要展开冗长的参数菜单即可完成高频操作。
*   **🔴🟡🟢 实体交通灯**：直观展示当前节点的逻辑控制状态（🟢 直通 / 🔴 拦截 / 🟡 手动），单击即可直接循环切换状态。
*   **流式自适应排版**：工具栏图标采用现代响应式设计，拉大或缩窄节点时，图标会自动调整间距（最小 1 像素间距，并配有 260px 宽度物理保护），点击判定区像素级随动。
*   **隐藏式输入端口优化**：移除了多余的输入文本描述，大幅节省节点内部排版空间，带来极致纯净的视觉感受。

#### 🛠️ 工具栏快捷动作详情
1.  **📁 选择收藏 (📁)**：点击调出文本收藏夹分类弹窗，一键将历史收藏提示词直接灌入当前文本框。
2.  **📋 一键粘贴 (📋)**：极速读取系统剪贴板文本并写入当前节点。
3.  **🗑️ 快速清空 (🗑️)**：一键将节点内的全部文本归零。
4.  **⭐ 极速收藏 (⭐)**：直接获取当前文本框的实时输入，唤起命名和分类选择弹窗，极速同步保存到系统的“文本收藏夹”统一数据库。
5.  **🚀 多选强化词 (🚀)**：不用在属性框内打字，点击直观展示多选强化词 preset 预设，勾选即时追加。
6.  **🔑 设定前缀 (🔑)**：快捷输入/修改固定前缀（Keyword）。
7.  **🎲 随机 SFW 模式 (🎲)**：快捷开关，一键开启或关闭收藏库的 SFW 提示词随机抽取模式。

### 3. 多图预览 (WuhuoMultiPreview)
一个增强版的预览节点，支持对比、历史回溯及快捷操作。

*   **双路输入**:
    *   `images`: 主图输入。
    *   `images_opt`: 对比图输入（可选），方便同时查看两组效果（如生图 vs 重绘）。
*   **交互操作 (高级)**:
    *   **双击节点**: 触发 **Queue Selected Output Nodes**（仅运行此节点及其上游）。
        *   *注*: 如果安装了 **rgthree** 插件，会自动调用 rgthree 的优化执行逻辑；否则使用内置的递归执行逻辑。
    *   **单击图片**: 进入大图预览模式。
    *   **ESC 键**: 在预览模式下退出预览。
    *   **D 键**: 鼠标悬浮在预览历史图片上时，按下 D 键即可直接在物理磁盘层面删除该张图片（带二次确认）。
*   **历史记录**:
    *   自动保存经过该节点的图片到 `custom_nodes/jdsc/data/history_images`。
    *   节点下方提供历史缩略图，点击即可在主视图回看旧图。
    *   **右键菜单**: 支持“打开保存目录”、“清空历史”等操作。
*   **动态布局**:
    *   根据图片数量自动调整网格（1x1, 1x2, 2x2 等）。
    *   拖拽改变节点大小时，图片会自动适应布局。

### 4. 空latent+ (WuhuoEmptyLatentVideo)
专为视频生成 (AnimateDiff, HunyuanVideo 等) 设计的 Empty Latent。

*   **核心优势**: **自动计算帧数**。你只需要输入视频时长（秒），节点会根据公式自动算出需要的 latent 帧数。
*   **参数**:
    *   `size_mode`: 预设比例 (16:9, 4:3, 9:16 等) 或自定义。
    *   `orientation`: 横屏/竖屏一键切换。
    *   `seconds`: 视频时长（秒）。
    *   `width/height`: 基础分辨率。
*   **输出**: 包含 Batch 维度 (T) 的 Latent，且宽高自动对齐到 8 的倍数。

### 5. 空latent+Qwen (WuhuoEmptyLatentQwen)
通用的 Empty Latent 增强版，适用于 FLUX, SDXL, Qwen 等文生图模型。

*   **参数**:
    *   `batch_size`: 生成数量。
    *   `size_mode`: 常用画幅比例预设。
    *   `width/height`: 分辨率（默认 1024x1024）。
*   **输出**: 标准 Latent Tensor `[B, C, H, W]`。

### 6. 忽略选择框 (WuhuoIgnoreGroup)
这是一个用于控制组 (Group) 状态的辅助节点，支持与 **文本传递 (TextGate)** 节点联动。

*   **基本功能**:
    *   `target_group`: 选择一个画布上的组 (Group)。
    *   `enable`: 开关。
        *   **ON (开启)**: 目标组内的所有节点设为 **Always (运行)**，组展开。
        *   **OFF (关闭)**: 目标组内的所有节点设为 **Never (静音/跳过)**，组折叠。

*   **🔗 自动化联动 (核心玩法)**:
    *   当画布上存在 **TextGate** 节点时，IgnoreGroup 会自动跟随 TextGate 的状态：
        *   **TextGate 🟢绿灯 / 🔴红灯**: 自动 **开启** IgnoreGroup (组运行)。
        *   **TextGate 🟡黄灯**: 自动 **关闭** IgnoreGroup (组静音)。
    *   **应用场景**:
        *   将耗时的“放大/重绘/人脸修复”模块放入一个组，并用 IgnoreGroup 控制。
        *   当你调试 Prompt (黄灯) 时，这些耗时模块会自动跳过，实现秒级出图。
        *   当你准备正式出图 (绿灯) 时，这些模块会自动恢复运行。
